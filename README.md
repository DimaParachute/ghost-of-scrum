# Ghost of Scrum Bot

Телеграм-бот — помощник для скрам-команды в рабочем чате.

## Возможности

- Напоминания о встречах: дейли по будням, еженедельные демо, двухнедельные ретро и т.п.
- Голосование за спринт (0–10) каждые 2 недели в пятницу второй недели спринта в 12:00 МСК. Считает среднее, показывает счётчик голосов, можно закрыть досрочно.
- Голосование по тестовой среде (0–5) сразу после, в 12:01 МСК. Голосуют зарегистрированные тестировщики, считается среднее.
- Случайный выбор фасилитатора дейли каждый будний день в 11:59 МСК. Если выбранный отказался — бот выбирает замену. Если список пуст — дейли ведёт скрам-мастер. Флоу можно отключить для чата — тогда дейли всегда ведёт скрам-мастер.
- Учёт отпусков: отпускники показываются в daily-сообщении, не попадают в случайный выбор фасилитатора, можно выгрузить годовой PDF-календарь.
- В пятницу в конце спринта вместо случайного фасилитатора тегается зарегистрированный скрам-мастер (он ведёт планирование в понедельник).
- Поздравления с днём рождения в 09:00 МСК. Каждый ставит свою дату сам или админ — в reply на сообщение нужного человека.

Бот работает в нескольких чатах одновременно: каждый рабочий чат регистрируется через `/setchat`, у каждого свои `team_size`, `sprint_start` и режим фасилитаторов. Двухнедельные события считаются от `sprint_start` этого чата и тикают каждые 14 дней — вручную начинать новый спринт не нужно.

## Локальный запуск (macOS / Linux)

Нужен Python 3.9+.

```bash
git clone <repo-url>
cd ghost-of-scrum-bot

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt    # либо см. список ниже
```

Для `/vacationpdf` в репе уже лежат TTF-шрифты с кириллицей в `assets/fonts/`. Если их удалить, бот попробует найти системный DejaVu/Arial.

Если `requirements.txt` ещё нет, поставить можно так:
```bash
pip install "python-telegram-bot[job-queue]" APScheduler tzdata reportlab pillow
pip freeze > requirements.txt
```

Создать `.env` в корне проекта:
```
TG_BOT_TOKEN=токен_от_BotFather
```

Запуск:
```bash
export $(grep -v '^#' .env | xargs)
python bot.py
```

## Настройки в `bot.py`

Глобальные:

| Переменная | Что | По умолчанию |
|---|---|---|
| `TIMEZONE` | Часовой пояс расписания | `Europe/Moscow` |
| `SPRINT_POLL_TIME` | Время голосования за спринт | `12:00` |
| `ENV_POLL_TIME` | Время голосования по тестовой среде | `12:01` |
| `DEFAULT_TEAM_SIZE` | Размер команды для нового чата | `6` |
| `DEFAULT_SPRINT_START_DATE` | Дата старта спринта для нового чата (`YYYY-MM-DD`) | `2026-04-29` |
| `DEFAULT_FACILITATORS_ENABLED` | Включён ли флоу фасилитаторов для нового чата | `True` |

`DEFAULT_TEAM_SIZE`, `DEFAULT_SPRINT_START_DATE` и `DEFAULT_FACILITATORS_ENABLED` — только дефолты при `/setchat`. Каждый чат хранит свои `team_size`, `sprint_start` и режим фасилитаторов. Размер команды и старт спринта меняются через `/setteamsize` и `/setsprintstart`, режим фасилитаторов — через `/disablefacilitators` и `/enablefacilitators`.

## Команды бота

Это то же, что показывает `/start` в Telegram.

```
👋 Привет! Я скрам-помощник команды.

Что я умею:
1) Напоминать о встречах (дейли, ретро и др.)
2) Раз в 2 недели присылать голосование за спринт (0–10) и считать среднее

📌 В каждом рабочем чате (если у тебя их несколько — у каждой команды свой)
   выполни /setchat — тогда я начну присылать туда голосование за спринт.
   Снять чат с роли рабочего: /unsetchat (для админов).
   Каждый чат имеет свои team_size, sprint_start и режим фасилитаторов.

🛎 Команды напоминаний:
/daily ЧЧ:ММ Текст — каждый будний день
/weekly ДЕНЬ ЧЧ:ММ Текст — раз в неделю
/biweekly ДЕНЬ ЧЧ:ММ Текст — раз в 2 недели (отсчёт от sprint_start)
ДЕНЬ: mon, tue, wed, thu, fri, sat, sun
/list — список напоминаний этого чата
/remove ID — удалить напоминание по ID

📊 Голосование за спринт (пт 12:00 МСК, раз в 2 недели):
/startpoll — запустить голосование вручную прямо сейчас
/closepoll — досрочно закрыть голосование и показать среднее

🧪 Оценка тестовой среды (пт 12:01 МСК, раз в 2 недели, голосуют тестировщики, считается среднее):
/registertester — добавить себя в список тестировщиков
/unregistertester — убрать себя из списка
/addtester (reply, для админов) — добавить того, на чьё сообщение отвечаешь

/closeenvpoll — досрочно закрыть голосование и показать среднее

🎲 Выбор фасилитатора дейли (по умолчанию каждый будний день в 11:15 МСК):
/joindaily — добавить себя в список фасилитаторов
/leavedaily — убрать себя из списка
/addfacilitator (reply, для админов) — добавить того, на чьё сообщение отвечаешь
/removefacilitator (reply, для админов) — удалить того, на чьё сообщение отвечаешь
/dailymembers — показать список
/picknow — выбрать фасилитатора прямо сейчас (для теста)
/setdailytime ЧЧ:ММ — время выбора фасилитатора

/registerscrum — зарегистрироваться скрам-мастером
/unregisterscrum — снять регистрацию
/scrum — показать зарегистрированного скрам-мастера
/setscrum (reply, для админов) — назначить того, на чьё сообщение отвечаешь
/unsetscrum (для админов) — снять текущего скрам-мастера

🎂 Дни рождения (поздравление в чат в 09:00 МСК):
/setbirthday ДД.ММ — поставить себе (или в reply на сообщение — поставить другому, для админов)
```

## Порядок настройки в чате

1. Добавить бота в рабочий чат, сделать админом.
2. Выполнить `/setchat` — чат добавится в список рабочих. Можно повторить в нескольких чатах для разных команд.
3. Списки фасилитаторов и тестировщиков можно сформировать двумя способами:
   - **Самообслуживание:** каждый пишет `/joindaily` или `/registertester`.
   - **Админом чата:** в ответ (reply) на сообщение нужного человека выполни `/addfacilitator` или `/addtester`. Удалить — `/removefacilitator` / `/removetester` тем же способом.
5. Завести нужные напоминания: `/daily 10:00 Дейли-стендап`, `/biweekly thu 15:00 Ретро` и т.п.

## Персистентность (state.json)

Всё пользовательское состояние хранится в JSON-файле рядом с `bot.py` (`state.json` по умолчанию, путь можно переопределить переменной окружения `STATE_PATH`). После каждой мутации файл атомарно перезаписывается. Сохраняется:

- `chat_configs` (список рабочих чатов и их `team_size`/`sprint_start`/режим фасилитаторов)
- `team_members`, `testers`, `scrum_masters`, `birthdays`, `vacations`
- активные голосования: за спринт (`polls`), тестовую среду (`env_polls`), активные сообщения «кто проводит дейли» (`daily_picks`)
- пользовательские напоминания (`/daily`, `/weekly`, `/biweekly`)

При старте бот загружает `state.json` (если есть), восстанавливает структуры в памяти и пересоздаёт все джобы пользовательских напоминаний. Кнопки на старых сообщениях с голосованиями продолжают работать после рестарта.

Файл небольшой (десятки KB) и читаемый — можно бэкапить копированием:
```
cp state.json state.json.$(date +%F).bak
```

В git файл не коммитится (`.gitignore`).

## Миграции схемы

В корне `state.json` хранится `"_version"`. Аддитивные изменения (новое поле, новый словарь) делаются без миграций — код читает с дефолтами. Для breaking-изменений (переименование, изменение типа) добавляется функция в словарь `MIGRATIONS` в `bot.py`; перед применением создаётся бэкап `state.json.v{N}.bak.json`. Если миграция падает — бот не стартует, чтобы не повредить файл.

## Известные ограничения

- Тег пользователя кликабельный, если у него есть `username` или он хотя бы раз взаимодействовал с ботом (после `/joindaily`, `/registertester` или `/setvacation` — взаимодействовал).
